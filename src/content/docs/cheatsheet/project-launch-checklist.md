title: "Чеклист: запуск нового проекта"
description: "Пошаговый чек-лист для запуска нового проекта — от идеи до первого релиза"
---

## Идея и валидация

- [ ] Сформулирована проблема, которую решает проект
- [ ] Определена целевая аудитория
- [ ] Проведены 5-10 проблемных интервью
- [ ] Изучены конкуренты (минимум 3)
- [ ] Сформулировано УТП (уникальное торговое предложение)
- [ ] Выбрана бизнес-модель и способ монетизации

## Инфраструктура

- [ ] Зарегистрирован домен
- [ ] Создан GitHub-репозиторий (или GitLab/Gitea)
- [ ] Настроен CI/CD (GitHub Actions)
- [ ] Выбран хостинг / PaaS
- [ ] Настроен мониторинг (Sentry, Grafana)
- [ ] Настроен Uptime-мониторинг (Better Uptime / Upptime)
- [ ] Настроен сбор аналитики

## Коммуникация

- [ ] Создан Telegram-канал проекта
- [ ] Создан email для поддержки
- [ ] Настроена транзакционная почта (Resend, Postmark)
- [ ] Описаны каналы связи для пользователей

## Разработка

- [ ] Определён стек технологий
- [ ] Создана структура проекта (монорепо/полирепо)
- [ ] Написан README.md с инструкцией по запуску
- [ ] Настроен линтер и форматтер
- [ ] Написаны первые тесты
- [ ] Настроен pre-commit хук
- [ ] Определён процесс релизов и версионирования

## Запуск

- [ ] Собрана лендинг-страница (или MVP)
- [ ] Описан Changelog
- [ ] Подготовлены социальные сети
- [ ] Написан анонс запуска
- [ ] Разосланы приглашения первым пользователям
- [ ] Настроена обработка обратной связи

## После запуска

- [ ] Настроены дашборды ключевых метрик
- [ ] Определён процесс работы с багами и фидбеком
- [ ] Настроены регулярные ретроспективы

## Инструменты для каждого этапа

| Этап | Инструменты |
|------|------------|
| Задачи | Linear, Jira, ClickUp |
| Документация | Notion, Obsidian, GitBook |
| Код | GitHub, GitLab |
| CI/CD | GitHub Actions, GitLab CI |
| Хостинг | Vercel, Railway, Fly.io, DigitalOcean |
| Мониторинг | Sentry, Grafana, Upptime |
| Коммуникация | Telegram, Slack |
| Email | Resend, Postmark, Unisender |
| Аналитика | Plausible, Umami, PostHog |
