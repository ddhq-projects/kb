## Что такое Marketing as Code

**Marketing as Code (MaC / MACC)** — подход, который применяет инженерные принципы (как в Infrastructure as Code) к маркетингу. Вместо ручного создания кампаний маркетологи задают цели (objectives), ограничения (guardrails) и данные, а AI-системы автономно генерируют, тестируют и оптимизируют кампании.

Ключевой сдвиг: от фиксированных if-then правил к **автономным циклам**, где AI оперирует миллионами динамических сегментов в реальном времени.

Это **парадигмальный уровень** по отношению к [MARKETING.md](./marketing-md.md) — MARKETING.md даёт формат конфигурации, а Marketing as Code описывает всю операционную модель: как агенты работают, какие роли выполняют, как строится инфраструктура автономного маркетинга.

## Сравнение с традиционной автоматизацией

| Аспект | Традиционная автоматизация | Marketing as Code |
|--------|---------------------------|-------------------|
| Решения | Статические human-правила | AI-цели + live-данные |
| Масштаб сегментации | 5–10 сегментов | Миллионы динамических сегментов |
| Данные | Синхронизированные снепшоты | Live enterprise data (zero-copy) |
| Роль человека | Строит каждый шаг | Задаёт цели/ограничения, проверяет результаты |
| Формат вывода | Читаемые человеком кампании | Machine-readable для AI-агентов |

## Ключевые принципы

### 1. Objectives & Guardrails

Вместо «сделай email-кампанию на тему X» — точные цели и ограничения:

- **Цели**: «+20% trials, CPA < $X»
- **Ограничения**: бренд-тон, бюджет, каналы, compliance

Агент сам решает *как* достичь цели в рамках guardrails.

### 2. Machine-Readable Signals

Структурированные данные о продукте для AI-агентов — не маркетинговые тексты, а факты: фичи, цены, аудитория, конкуренты. Агенты сами переводят это в messaging.

### 3. Unified Channels

Email, ads, web, push — как единая координированная система, а не изолированные каналы. Агент координирует сообщения across channels.

### 4. Autonomous Loops

Цикл: **генерация → тест → оптимизация → перераспределение бюджета**. Без участия человека между итерациями.

### 5. Human-in-the-Loop

Человек определяет стратегию и проверяет ключевые решения. AI исполняет. Не «AI вместо маркетолога», а «AI как исполнитель при маркетологе-стратеге».

## 5 ролей AI-агентов в маркетинге

Классификация от Mayfield (венчурные инвесторы Kana Intelligence):

| Роль | Что делает |
|------|-----------|
| **«Busy middle»** | Генерация и тестирование вариантов креативов, мониторинг, остановка проигравших кампаний |
| **Cross-channel координатор** | Каналы не общаются друг с другом — агенты координируют: «этому пользователю уже показали ad, email отложим на завтра» |
| **Экспериментальная инфраструктура** | Непрерывное A/B-тестирование, а не поквартальное |
| **Персонализация в масштабе** | Миллионы микро-сегментов без обвалов performance |
| **Ранние предупреждения** | Creative fatigue, CAC creep, brand drift — AI замечает до того, как станет поздно |

## Маркетолог будущего

По версии Mayfield, маркетолог будущего:

- **Мыслит системами** — пишет цели, а не задачи
- **Проверяет решения**, а не драфты
- **Управляет портфелем агентов**, а не календарём задач
- **Команда:** 15 человек + fleet of agents > 50 человек manual

## Практические кейсы

### Claude Code для маркетинга (Firecrawl)

SEO-лид Firecrawl делится реальным опытом ежедневного использования Claude Code:

- **Кастомные агенты для трекинга упоминаний** — Reddit, Hacker News, Twitter
- **Авто-обновление блог-постов** — Ahrefs выявляет «протухающий» контент → Slack-уведомление → Claude обновляет
- **Агент внутренней перелинковки** — заменил SaaS-инструмент за $30/мес
- **Скрейпинг конкурентов** через Firecrawl API
- **Оптимизация изображений, SEO-аудиты, AI visibility tracking**

Ключевой инсайт: **«AI does tedious stuff so I focus on what matters»**. Маркетологу полезно знать базу: HTML, CSS, JS, tech stack, SEO fundamentals — но Claude Code не заменяет стратегию, он её исполняет.

### Автономные кампании (Pushwoosh)

Результаты внедрения AI-автоматизации:

- **3x open rates** от AI-определения оптимального времени отправки
- **+40% revenue за 90 дней**
- AI генерирует целые стратегии (например, рождественские email- journeys) за минуты

Полный lifecycle автоматизации: content generation → personalization → optimal timing → segmentation.

### IBM Research

72% AI-инициатив в маркетинге проваливаются без глубокой интеграции в workflows. Лидеры, которые интегрируют AI в процессы, получают **+60% к эффективности**. Вывод: AI в маркетинге — не про инструменты, а про перестройку процессов.

## Marketing as Code и MARKETING.md

| Аспект | Marketing as Code (парадигма) | MARKETING.md (формат) |
|--------|------------------------------|----------------------|
| Уровень | Операционная модель | Конфигурационный файл |
| Вопрос | «Как AI управляет маркетингом?» | «Какие правила у бренда?» |
| Аналогия | Infrastructure as Code | docker-compose.yml |
| Кто автор | Marchcroft, Mayfield, Firecrawl | Benjamin Koh (PayPal) |

MARKETING.md — это файл конфигурации для Marketing as Code. MaC описывает *как система работает*, MARKETING.md — *по каким правилам*.

## Когда внедрять

Marketing as Code оправдан когда:

- Объём маркетинговых операций перерос ручное управление
- Есть data-инфраструктура (CRM, analytics, CDP)
- Команда готова перейти от «делаем кампании» к «управляем системой»
- Есть техническая компетенция или партнёрство CMO+CTO

Для малого бизнеса с 1–2 кампаниями в месяц overhead от автономных систем может превышать выгоду. Начинать стоит с MARKETING.md как конфига — это даёт консистентность без сложной инфраструктуры.
