## Page Format

Every wiki page (except index.md and log.md) starts with Starlight-compatible frontmatter:

```yaml
---
title: "Page Title"
description: "One-line summary of this page"
---
```

Starlight automatically reads `title` and `description` for:
- Page heading
- Meta description tag
- Site search indexing
- Pagefind / Algolia

**Important:** Do NOT start pages with an `# H1` heading that duplicates the frontmatter `title`. Starlight already renders the `title` as the page's H1. Start content directly with `##` level headings.

### Источники (Source Attribution)

Каждая wiki-страница должна заканчиваться секцией **«Материалы и источники»**:

```md
## Материалы и источники

- [Оригинальная статья](https://example.com/...)
```

- Ссылка на оригинальный URL (из frontmatter `source` raw-файла)
- Если страница собрана из нескольких источников — перечислить все

## Content Strategy

### Evergreen-контент

Приоритет — **evergreen-контент**: статьи, которые остаются актуальными месяцами и годами.

- Избегать срочных новостей, временных акций и материала с коротким сроком годности
- Фокус на фундаментальных вопросах, которые люди ищут снова и снова

### Что ищут пользователи

При выборе тем для новых страниц ориентироваться на **регулярный поисковый спрос**:

- Топики, которые стабильно ищут в поиске (Яндекс, Google) — не разовые тренды
- Проблемы и задачи, с которыми пользователи сталкиваются циклически (выбор CRM, настройка аналитики, сравнение сервисов, налоги и т.д.)
- «Вечнозелёные» запросы: как сделать X, что лучше X или Y, обзор X, инструкция по X
- При планировании контента можно проверять поисковый спрос через веб-поиск

## Content Focus

### Приоритет сервисов

При работе с сервисами и приложениями:
- **Приоритет №1** — российские сервисы (популярные в РФ): Яндекс, VK, Сбер, Тинькофф, Касперский, МТС, Билайн, Roo, Ростелеком и др.
- **Приоритет №2** — открытый софт (FOSS): бесплатные, с открытым исходным кодом (opensource), self-hosted решения.
- **Приоритет №3** — зарубежные сервисы: учитываем, но акцент ниже. Упоминать только если нет достойной РФ/FOSS-альтернативы или если сервис объективно лидирует в нише.

Особенно эти приоритеты важны когда собираем сводные таблицы, подборки и рейтинги.

### Области покрытия

База знаний фокусируется на:
- **Сервисы для бизнеса** — CRM, ERP, финансовые инструменты, аналитика, маркетинг
- **Образ жизни** — продуктивность, саморазвитие, здоровье, умный дом, досуг
- **AI-инструменты** — LLM, генерация контента, AI-агенты, AI для кода

- **IT-инструменты** — разработка, DevOps, безопасность, хостинг, CI/CD, мониторинг

Каждая страница должна содержать:
1. Что это за сервис/инструмент
2. Ключевые возможности
3. Цены и тарифы (если применимо)
4. Сравнение с аналогами
5. Практические сценарии использования

## Raw File Safety

- Путь для raw-файлов: `raw/YYYY/MMDD/filename.md` (без префикса `kb-ddpa/`)
- **Перед `rm -rf` всегда делай `ls`** для проверки содержимого
- Если `mv` завершился ошибкой — проверь, где остался файл, прежде чем удалять что-либо
