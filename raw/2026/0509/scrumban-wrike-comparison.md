## Key Differences

| Feature | Scrum | Kanban | Scrumban |
|---------|-------|--------|----------|
| Workflow | Fixed sprints (1-4 weeks) | Continuous flow | Continuous with optional triggers |
| Roles | Product Owner, Scrum Master, Developers | None required | Flexible/optional |
| Planning | Sprint-based | On-demand | Trigger-based |
| WIP Limits | Implicit (sprint capacity) | Explicit per column | Explicit per column |
| Changes | Discouraged mid-sprint | Allowed anytime | Allowed anytime |
| Board | Resets per sprint | Persistent | Persistent |

## When to Use Each

**Scrum** — complex, long-term projects needing predictability, regular feedback, and structure. Cross-functional teams handling dynamic work.

**Kanban** — unpredictable tasks, ongoing maintenance, support, fluctuating priorities (IT operations, small teams).

**Scrumban** — teams transitioning between frameworks, handling mixed planned and urgent work, fast-paced settings like startups with evolving requirements and concurrent projects.
