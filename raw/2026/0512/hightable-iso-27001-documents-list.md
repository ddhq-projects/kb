# ISO 27001 Template Documents — Ultimate Guide

**Source:** https://hightable.io/iso-27001-documents/
**Fetched:** 2026-05-12

The basic foundation of any information security management system is having documentation in place and making sure you have the required, mandatory documents. If you do not have the mandatory documents, you won't pass your ISO 27001 Certification.

## Complete List of ISO 27001 ISMS Templates and Document Assets

| Document Template | Purpose |
|---|---|
| ISO 27001 Organisation Overview Template | Provides a high-level articulation of the organisation's identity |
| ISO 27001 Context of Organisation Template | Determines internal/external issues and stakeholder requirements |
| ISO 27001 Scope Document Template | Formally records the boundaries of the ISMS, including exclusions |
| ISO 27001 Legal Register Template | Tracks statutory, regulatory, and contractual obligations |
| ISO 27001 Physical Asset Register Template | Maintains a record of hardware devices that store/process/transmit sensitive data |
| ISO 27001 Statement of Applicability Template | Mandatory documentation identifying which Annex A controls are implemented and why |
| ISO 27001 Competency Matrix Template | Tracks staff skills and training requirements for ISMS compliance |
| ISO 27001 Information Classification Template | A visual summary and 'cheat sheet' for staff regarding data handling and classification |
| ISO 27001 Data Asset Register (ROPA) | Aligns ISO 27001 with GDPR through a detailed Record of Processing Activities |
| ISO 27001 Audit Plan Template | Used to schedule internal and external audit cycles |
| ISO 27001 Audit Report and Worksheets | Detailed worksheets for auditing ISMS clauses and Annex A controls |
| ISO 27001 Risk Management Process Template | Defines the step-by-step procedure for identifying, evaluating, and treating risks |
| ISO 27001 Risk Register Template | The central repository for managing information security risks and treatment plans |
| ISO 27001 Incident & Corrective Action Log | Records security incidents and the resulting improvements |
| ISO 27001 Supplier Register Template | Manages third-party risks through contract tracking and security assurance |
| Management Review Meeting Agenda | Structured agenda for leadership oversight |
| Information Security Document Tracker | Tracks owners, version control, and review status of the entire ISMS document set |
| ISO 27001 RASCI Accountability Template | Maps responsibility and accountability for Annex A controls across the organisation |
| Business Impact Analysis Template | Analyses operational disruptions to set recovery time objectives (RTO) |
| Business Continuity Objectives & Strategy | Documents the high-level approach to maintaining business resilience |
| Business Continuity Plan Template | The actionable manual for recovering operations after a significant incident |

## Mandatory vs Recommended Documents

| Document Name | Audit Status | ISO 27001 Ref. |
|---|---|---|
| ISMS Scope Statement | **Mandatory** | Clause 4.3 |
| Statement of Applicability (SoA) | **Mandatory** | Clause 6.1.3 |
| Risk Assessment & Treatment | **Mandatory** | Clause 6.1.2 |
| Information Security Policy | **Mandatory** | Clause 5.2 |
| Internal Audit Results | **Mandatory** | Clause 9.2 |
| Mobile Device Policy | Recommended | Annex A 6.7 |
| Access Control Policy | Recommended | Annex A 5.15 |
| Physical Asset Register | Recommended | Annex A 5.9 |
| Business Continuity Plan (BCP) | Recommended | Annex A 5.29 |

## The 11 New ISO 27001:2022 Controls: Documentation Requirements

The 2022 update introduced 11 new controls that auditors now scrutinize heavily. To satisfy a 2026 audit, you cannot simply "tweak" old documents; you need specific procedures and records for these new thematic areas.

### ISO 27001:2022 Clause to ISMS Template Mapping Table (partial)

| Clause | Control | Template |
|---|---|---|
| 4.1 | Understanding the organisation and its context | Context of Organisation |
| 4.2 | Understanding the needs and expectations of interested parties | Context of Organisation |
| 4.3 | Determining the scope of the ISMS | Documented ISMS Scope |
| 5.1 | Leadership and commitment | Organisation Overview + Management Review |
| 5.2 | Policy | Information Security Policy |
